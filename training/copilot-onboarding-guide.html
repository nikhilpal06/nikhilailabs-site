The big idea

A copilot is a system, not a prompt

  1. ✓Method is enforced. Phase gates and guardrails won't let you skip steps or overclaim a root cause.
  2. ✓The numbers are correct. Statistics are computed by a tested engine — not guessed by a chatbot.
  3. ✓It coaches you. It tells you what's missing and what to do next — not just what you asked.
  4. ✓You stay in control. AI is advisory; a qualified human makes every decision.
Remember: the copilot does the structure, the math, and the discipline. You bring the judgment, the context, and the decision.
PhilosophyHuman-in-the-loop by design

Good habits

Five habits that apply everywhere

Tool 1 · How to run it

Diagnose a problem in 60 seconds

  1. 1From the homepage, click Analyze my problem (or go to /assessment).
  2. 2Describe the problem in plain English, or click an example chip to auto-fill.
  3. 3Click Analyze My Problem → your instant result appears in seconds.
  4. 4Read the cards: problem type, refined statement, likely cause areas, draft 5-Why, readiness score, missing info.
  5. 5Explore the Problem Intelligence Map — the orange node is your recommended copilot.
  6. 6Click ✨ Deepen with AI for a richer 5-Why and a human practitioner take.
  7. 7Click Continue in [copilot] to start the real work — or Email me this assessment.
It's a triage, not a verdict. Use it to choose the right tool and see what evidence you still need — never as a final root cause.

Tool 3 · Coaching

A3 Coach

When to use it: any structured problem you want to solve and learn from — building a rigorous A3 while an AI coach sharpens your thinking.

  1. 1Start a new A3.
  2. 2Work the boxes in order: Background → Current Condition → Goal → Root-Cause Analysis → Countermeasures → Plan → Follow-up.
  3. 3Use the logic map (5-Why / Fishbone) to structure root cause; the coach challenges weak links.
  4. 4Respond to the coaching prompts — it pressure-tests your logic, it doesn't just fill the form.
  5. 5Check leadership-review readiness, refine, then save / export your A3.
Pro tip: let it challenge you. The value is better thinking and stronger problem solvers — not a faster form.
A3 Coacha3.nikhilailabs.com
Tool 4 · Events

Kaizen Event Copilot

When to use it: planning, running, documenting, and sustaining a rapid-improvement (Kaizen) event.

  1. 1Readiness scoring — answer the readiness questions; get a go / no-go score.
  2. 2Charter & agenda — generate the event charter, scope, and a day-by-day agenda.
  3. 3Analysis — walk the 8 wastes and root cause on the target process.
  4. 4Countermeasures — generate ideas and rank them on the impact / effort matrix.
  5. 5Action & control plan + a 30/60/90-day sustainment plan → export the Kaizen report.
Pro tip: do the readiness step honestly — a weak charter is the #1 reason events fail to stick.
Kaizen Event Copilotkaizen.nikhilailabs.com
Tool 5 · Flow

Flow Intelligence Copilot

When to use it: mapping a value stream or process and finding the bottlenecks, waste, and highest-impact improvements.

  1. 1New map → pick a type (value stream / process / metric flow / future state).
  2. 2Drag node types onto the canvas (suppliers, processes, queues, inspections, rework, kaizen burst…) and connect them.
  3. 3Enter metrics on each step: cycle time, wait, WIP, first-pass yield, defect rate, changeover, uptime.
  4. 4Click Analyze → AI finds bottlenecks, classifies the 8 wastes, and computes flow efficiency / rolled throughput yield.
  5. 5Review the readiness / performance / confidence scores, then the 30/60/90 roadmap and future-state ideas.
  6. 6Import an existing map (Miro, Lucid, CSV, PDF, screenshot) with AI extraction + a review step; export the branded PDF.
Garbage in, garbage out: the analysis is only as good as the metrics — capture real cycle and wait times.

Tool 6 · The statistics engine

Run a real analysis

  1. 1Open the Statistics tab. Unsure which test? Use "Which statistical tool should I use?" (Test or Control-chart mode).
  2. 2Choose an analysis: Pareto, Cp/Cpk, I-MR, X̄-R, p/np/c/u, run chart, t-test, ANOVA, chi-square, regression.
  3. 3Paste or upload your data (CSV/Excel); enter spec limits where the analysis needs them.
  4. 4Click Run analysis → read the result, chart, and the plain-English interpretation (what it says / what it does NOT prove / next step).
  5. 5Save to project (feeds the AI review & exports); export PNG / CSV per analysis.
Built-in guardrail: it refuses to run the wrong analysis on the wrong data (e.g. capability on category counts) and never declares a final root cause.

Trust & compliance

How the copilots stay trustworthy

  1. ✓Human-in-the-loop by design. The AI structures and suggests; you approve and decide.
  2. ✓No overclaiming. It separates facts from assumptions and won't assert a final root cause or significance without evidence.
  3. ✓Right analysis only. It blocks statistically invalid analyses instead of returning a confident-but-wrong answer.
  4. ✓Regulated-ready. Audit-friendly outputs and disclaimers — "do not use as final GMP documentation without QA review."
  5. ✓Your data stays in the platform. The public Diagnostic stores no sensitive text unless you consent.
GuardrailsRegulated by default
FAQ & troubleshooting

Common questions

I don't see a copilot

Access is per email. Ask your admin to grant it on the Access page.

The AI seems slow

Deeper analyses run in the background — give it a moment to finish.

My analysis was blocked

That's a guardrail. Check your data type matches the analysis (e.g. continuous vs. counts).

Did I lose my work?

Use ☁ Save for the cloud copy; your live draft autosaves in the browser.

Can I import existing work?

Yes — upload a charter, deck, CSV or PDF; review the AI's extraction before it loads.

Is it accurate?

Statistics are computed by a tested engine; AI text is advisory — always verify before acting.
